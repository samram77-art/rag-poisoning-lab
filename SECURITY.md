# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest (main) | ✅ |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

### How to Report

1. Email: **samsonram@hotmail.com**
2. Subject line: `[SECURITY] <repo-name> — Brief description`
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fix (optional)

### Response Timeline

| Stage | Timeline |
|-------|----------|
| Acknowledgement | Within 48 hours |
| Initial assessment | Within 7 days |
| Fix or mitigation | Within 30 days (critical), 90 days (others) |
| Public disclosure | After fix is released |

### Scope

This repository is a **security research and education tool**. It is designed to demonstrate vulnerabilities — not to be deployed in production. Please ensure you only use this software on systems you own or have explicit written permission to test.

### Recognition

Responsible disclosures will be credited in the project CHANGELOG unless you prefer to remain anonymous.
